安装

前提条件：

  • curl or installed wget
  • yarn
  • docker
  • git
npx @backstage/create-app@latest

运行后台应用

安装完成后，您可以转到应用程序目录并 启动应用。

该命令将同时运行前端和后端 在同一窗口中分隔进程（命名和 ）。前端[0] 后端[1]

cd my-backstage-app
yarn dev

插件安装

kubernetes

添加 Kubernetes 前端插件

将 Kubernetes 前端插件依赖添加到您的 Backstage 应用。

# From your Backstage root directory
yarn add --cwd packages/app @backstage/plugin-kubernetes

安装软件包后，您需要在应用程序中导入插件 通过将“Kubernetes”选项卡添加到相应的实体目录页面。

packages/app/src/components/catalog/EntityPage.tsx

import { EntityKubernetesContent } from '@backstage/plugin-kubernetes';

// You can add the tab to any number of pages, the service page is shown as an
// example here
const serviceEntityPage = (
  <EntityLayout>
    {/* other tabs... */}
    <EntityLayout.Route path="/kubernetes" title="Kubernetes">
      <EntityKubernetesContent refreshIntervalMs={30000} />
    </EntityLayout.Route>
  </EntityLayout>
);

添加 Kubernetes 后端插件

将 Kubernetes 后端插件依赖添加到您的 Backstage 应用。

# From your Backstage root directory
yarn add --cwd packages/backend @backstage/plugin-kubernetes-backend

创建一个 packages/backend/src/plugins/kubernetes.ts 文件，然后 添加以下内容：

import { KubernetesBuilder } from '@backstage/plugin-kubernetes-backend';
import { Router } from 'express';
import { PluginEnvironment } from '../types';
import { CatalogClient } from '@backstage/catalog-client';

export default async function createPlugin(
  env: PluginEnvironment,
): Promise<Router> {
  const catalogApi = new CatalogClient({ discoveryApi: env.discovery });
  const { router } = await KubernetesBuilder.createBuilder({
    logger: env.logger,
    config: env.config,
    catalogApi,
    permissions: env.permissions,
  }).build();

}

配置

在app-config.yaml中添加相应配置

kubernetes:
  serviceLocatorMethod:
    type: 'multiTenant'
  clusterLocatorMethods:
    - type: 'config'
      clusters:
        - url: http://127.0.0.1:9999
          name: minikube
          authProvider: 'serviceAccount'
          skipTLSVerify: true
          skipMetricsLookup: true
          serviceAccountToken: ${K8S_MINIKUBE_TOKEN}

toolbox

yarn add --cwd packages/app @drodil/backstage-plugin-toolbox

编译打包

主句执行以下命令，进行预编译

# From your Backstage root directory
yarn install --frozen-lockfile

# tsc outputs type definitions to dist-types/ in the repo root, which are then consumed by the build
yarn tsc

# Build the backend, which bundles it all up into the packages/backend/dist folder.
yarn build:backend

主机构建完成后，即可构建映像。使用 创建新应用程序时，包括以下内容：

packages/backend/Dockerfile目录下有一份Dockerfile，

需要修改基础镜像 node:16-bullseye-slim ==» node:16.19.1-bullseye-slim

需要使用backstage的根目录作为构建上下文执行，以访问需要的文件yarn.lock,package.json

FROM node:16.19.1-bullseye-slim

# Install sqlite3 dependencies. You can skip this if you don't use sqlite3 in the image,
# in which case you should also move better-sqlite3 to "devDependencies" in package.json.
RUN --mount=type=cache,target=/var/cache/apt,sharing=locked \
    --mount=type=cache,target=/var/lib/apt,sharing=locked \
    apt-get update && \
    apt-get install -y --no-install-recommends libsqlite3-dev python3 build-essential && \
    yarn config set python /usr/bin/python3

# From here on we use the least-privileged `node` user to run the backend.
USER node

# This should create the app dir as `node`.
# If it is instead created as `root` then the `tar` command below will fail: `can't create directory 'packages/': Permission denied`.
# If this occurs, then ensure BuildKit is enabled (`DOCKER_BUILDKIT=1`) so the app dir is correctly created as `node`.
WORKDIR /app

# This switches many Node.js dependencies to production mode.
ENV NODE_ENV production

# Copy repo skeleton first, to avoid unnecessary docker cache invalidation.
# The skeleton contains the package.json of each package in the monorepo,
# and along with yarn.lock and the root package.json, that's enough to run yarn install.
COPY --chown=node:node yarn.lock package.json packages/backend/dist/skeleton.tar.gz ./
RUN tar xzf skeleton.tar.gz && rm skeleton.tar.gz

RUN --mount=type=cache,target=/home/node/.cache/yarn,sharing=locked,uid=1000,gid=1000 \
    yarn install --frozen-lockfile --production --network-timeout 300000

# Then copy the rest of the backend bundle, along with any other files we might want.
COPY --chown=node:node packages/backend/dist/bundle.tar.gz app-config*.yaml ./
RUN tar xzf bundle.tar.gz && rm bundle.tar.gz

CMD ["node", "packages/backend", "--config", "app-config.yaml"]

docker打包命令

docker image build . -f packages/backend/Dockerfile --tag backstage
